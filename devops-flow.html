Monitoreo

Observabilidad CloudWatch + SNS — todo lo que ocurre tras el deploy

📊
CloudWatch Dashboard
online-store-<env>
4 widgets: Requests · Error rate (4xx/5xx) · Bytes downloaded · Ejecuciones de pipeline.
🚨
CloudWatch Alarms
5xxErrorRate > umbral · crítica
4xxErrorRate > umbral · informativa
Requests = 0 · solo prod, detecta outages
📨
SNS + CodeStar Notifications
Topic online-store-alerts-<env>
Eventos del pipeline: failed, succeeded, manual-approval-needed
Suscripción opcional por email (alerts_email).
📁
CloudFront Access Logs
Bucket online-store-cf-logs-<env>-<suffix>
Cifrado AES256 · retención 30d staging / 90d prod.
Listo para análisis con Athena.
🛡️
Auditoría de dependencias
npm audit --audit-level=high corre en cada build.
Vulnerabilidades altas/críticas en prod-deps detienen el pipeline.
📈
Cobertura de tests
Threshold definido en karma.conf.js.
Report cobertura.xml publicado a CodeBuild Reports en cada ejecución.
🔔 Flujo de una alerta crítica (ej. error 5xx tras deploy)
CloudFront detecta 5xx → Alarm *-cf-5xx-prod dispara → SNS publica al topic → 📧 Email al equipo DevOps
